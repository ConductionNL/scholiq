## Summary

Migrates all `x-openregister-notifications` blocks in `lib/Settings/scholiq_register.json` from the legacy dialect to the verified OpenRegister notification-engine dialect. Ten schemas are covered (19 notification rules total). The legacy dialect used engine-unsupported keys (`channel`, `recipient`/`recipientField`/`recipientFromTenantRole`, `@self.`, `lifecycleEnter`, boolean `calculated`, `idempotencyKey`, `alsoDispatchLifecycle`, `event`, `template`) that were silently ignored by the engine; all blocks now use only `trigger.type` / `channels[]` / `recipients[]` / inline `subject{nl,en}`. Two non-numeric field-change rules (`Regulation.coverageDropped`, original `AttendanceThreshold` boolean) and two schema rule sets (`GradeEntry`, `LearningPlan`) are deferred and documented in Caveats per the spec.

## Changes
- `lib/Settings/scholiq_register.json` — rewrote all 10 `x-openregister-notifications` blocks (Credential, AiFeature, Course, Lesson, Enrolment, Regulation, ExcuseRequest, AttendanceThreshold, DataExchangeJob, GradeNotification) from legacy to verified engine dialect; fixed GradeNotification.gradePublished trigger type from non-verified `"create"` to verified `"created"`
- `openspec/changes/scholiq-notifications/specs/spec.md` — added acceptance criteria spec file

## Test Coverage
No PHP or Vue code was changed. The JSON is validated by `python3 -m json.tool` and all 19 rules were verified against the verified-dialect constraints (trigger types, recipient kinds, subject{nl,en}, no legacy keys). All 14 hydra gates pass.

## Caveats
- `Regulation.coverageDropped` (ragStatus → "red", non-numeric) and original `AttendanceThreshold.onThresholdCrossed` (boolean `isThresholdCrossed`) are held pending `notification-updated-field-change-condition`; AttendanceThreshold ships as numeric `calculatedChange` on `unexcusedLesuren ≥ 16` instead.
- `GradeEntry.gradePublished` (parent fan-out via GradeRollupHandler) and `LearningPlan.*` (multi-signer expression conditions) are removed; GradeNotification.gradePublished (the append-only notification record created by GradeRollupHandler) remains with `trigger.type: "created"`.
- `idempotencyKey`/`alsoDispatchLifecycle` have no verified-dialect equivalent; one task item remains open pending confirmation from the engine owner.
