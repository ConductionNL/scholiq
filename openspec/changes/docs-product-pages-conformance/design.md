## Context

Scholiq's docs site at `docs/` is built on `@conduction/docusaurus-preset ^2.6.1` (already current — no bump needed). The site has real content (7 root MDs totalling ~2,261 lines, 11 tutorials with screenshots from PR #72) but is structurally non-conformant: wrong folder names, no canonical taxonomy, 74+ em-dash violations, no `installation.md`, no Redocusaurus, and no Dutch locale scaffold. The canonical structure is defined by ADR-030 and demonstrated by the openregister/docs reference implementation and the shillinq pilot (PR #83).

Constraints:
- Docusaurus `docs.path: './'` (root of docs/) with `exclude: ['**/node_modules/**', 'src/**']` — autogenerated sidebar will pick up new folders automatically
- Images in tutorials reference relative paths like `./01-first-launch-01.png` — these must travel with their MD files during the tutorials→user-guide rename
- `i18n/nl/` locale cannot be re-enabled until translated markdown exists; the SSR escape hatch (disable `nl` if build fails) is documented in the config comment

## Goals / Non-Goals

**Goals:**
- Bring docs/ into full conformance with the canonical product-pages spec
- Zero em-dashes remaining in docs/ after this PR (closes #76)
- Build passes (`npm run build` exits 0) with `en` locale at minimum
- Redocusaurus mounted at `/api` with a valid placeholder OAS 3.0 stub
- Dutch locale scaffolded and re-enabled (with escape hatch if SSR breaks)

**Non-Goals:**
- Translating any content to Dutch (tracked via #74)
- Authoring real UseCases/ or Integrations/ content (tracked via #73)
- Authoring the real OpenAPI spec (tracked via #75)
- Rewriting any long-form analysis content (only mechanical em-dash fix)

## Decisions

**D1: `FEATURES.md` → `Features/features.md` (single file, not split)**
The 38KB FEATURES.md is a rich market-analysis document — splitting it into per-feature pages is Tier-3 content authoring. This PR moves it as-is; the autogenerated sidebar surfaces it under the Features section. Future PRs may split it.

**D2: `USER-GUIDE.md` → `user-guide/index.md`**
The root USER-GUIDE.md is a high-level overview page, not a tutorial. Moving it as `user-guide/index.md` gives the user-guide section a landing page without requiring content changes.

**D3: `tutorials/` → `user-guide/` via `git mv`**
`git mv docs/tutorials docs/user-guide` preserves git history for all 11 tutorial files. The `_category_.json` files travel with the directory and must have their `label` and `description` updated from "Tutorials" to "User Guide".

**D4: Em-dash strategy — `' — '` (space–em-dash–space) → `', '`**
The dominant pattern is a spaced em-dash used as a clause separator. Replacing with `, ` is the correct English-prose fix. A small number of em-dashes appear inside table cells as short separators (`— ` at line start) or in JSON category descriptions; these also get replaced. After the sweep, re-run `git grep -E '—' docs/` to verify zero remaining.

**D5: Redocusaurus v2 + placeholder OAS stub**
`redocusaurus@^2.0.0` is the version used by openregister (reference implementation). The placeholder `static/oas/scholiq.json` is a minimal OAS 3.0.3 stub with the scholiq title/version/description. This lets the `/api` route render without build errors. Real spec authorship is #75.

**D6: Re-enable `nl` locale with escape-hatch comment**
Adding `nl` to `locales` scaffolds the locale dropdown and the `i18n/nl/` directory structure. If SSR breaks on any doc page during `npm run build`, the escape hatch is to revert `locales: ['en', 'nl']` back to `locales: ['en']` with a `// TODO #74` comment. The `i18n/nl/docusaurus-plugin-content-docs/current/` directory is created empty — Docusaurus tolerates an empty NL tree as long as the EN source exists.

## Risks / Trade-offs

- **SSR on nl locale** → Escape hatch: revert locales to `['en']` with comment if build fails. The `i18n/nl/` scaffold remains committed regardless.
- **Screenshot paths after `tutorials/` rename** → Tutorial MDs use relative paths (`./filename.png`). `git mv` moves the screenshots directory too. Verify with `npm run build` that no broken-image warnings appear.
- **Sidebar ordering** → Moving files into new folders changes autogenerated sidebar order. Add `sidebar_position` frontmatter to key files (intro.md stays at 1; installation.md at 2) to anchor the top of the sidebar.
- **Internal links in intro.md** → Three links reference old paths (`./ARCHITECTURE`, `./FEATURES`, `./DESIGN-REFERENCES`). These must be updated to the new canonical paths during the move step.

## Migration Plan

1. `git mv docs/tutorials docs/user-guide` — preserves history
2. Update `_category_.json` label/description in user-guide/
3. `git mv docs/FEATURES.md docs/Features/features.md` (create Features/ dir first)
4. `git mv docs/ARCHITECTURE.md docs/Technical/architecture.md` (create Technical/ dir first)
5. `git mv` remaining root MDs to Technical/
6. `git mv docs/USER-GUIDE.md docs/user-guide/index.md`
7. Create stub `UseCases/index.md` and `Integrations/index.md`
8. Create `installation.md`
9. Em-dash sweep: Edit replace_all on each affected file
10. Fix internal links in `intro.md`
11. Update `docs/package.json` — add redocusaurus
12. Update `docs/docusaurus.config.js` — Redocusaurus plugin, navbar, locales
13. Create `docs/static/oas/scholiq.json`
14. Create `docs/i18n/nl/docusaurus-plugin-content-docs/current/` (empty)
15. `npm install --legacy-peer-deps && npm run build` — verify exit 0

Rollback: revert the branch; no database or application changes to undo.
