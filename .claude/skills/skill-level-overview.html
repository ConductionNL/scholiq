Maturity Levels

L1 Anatomy The skill has a valid SKILL.md with frontmatter (name, description, metadata), numbered self-contained steps, and explicit guardrails defining what it must never do.
L2 Triggering Auto-detected: SKILL.md is under 500 lines and the description field is present. Full criteria: description written in third person, front-loads key use case in 250 chars, uses specific trigger terms, and large blocks extracted to subfolders via progressive disclosure.
L3 Patterns Auto-detected: at least one common pattern (model guard, AskUserQuestion, quality gates, or subfolder references in SKILL.md) AND at least one supporting subfolder (examples/, references/, or templates/). Full criteria: built on proven foundations with consistent patterns and standards references.
L4 Domain Contains business-specific knowledge unique to this project — Dutch government standards (BIO2, GEMMA, NLGov API rules), Nextcloud app patterns, OpenSpec workflow integration, and project architecture. Could not be dropped into a different project and work as-is.
L5 Measurement Auto-detected: 3+ eval scenarios, 10+/10+ trigger tests, last_validated is non-null in evals.json, and both grading.json and timing.json present under the evals/ folder (timing.json proves evals were actually executed, not just simulated). Full criteria also requires: baseline measured without the skill and at least one improve cycle completed.
L6 Self-improvement Captures learnings during execution in a learnings.md file with dated, atomic entries. The skill includes a "capture learnings" step and consolidates insights back into SKILL.md guardrails and rules after ~80–100 entries.
L7 Orchestration Spawns and coordinates multiple sub-agents, or is part of a defined workflow chain with explicit handoff points. Uses isolated execution contexts where needed, supports parallel execution, and has autonomous operation capability for a defined scope.

Updating

Run bash update-skill-overview.sh to auto-update:

  • Maturity levels (L1, L2, L3, L5, L6, L7 auto-detected from SKILL.md)
  • Line counts and color classes
  • L5/L6/L7 struct flags (orange ring detection)
  • Skill descriptions (synced from SKILL.md frontmatter, truncated at 150 chars)
  • Skill name links (new rows link to SKILL.md)
  • New skill rows (auto-added when a new skill directory is detected)
  • Orphan detection (warns when a row's skill directory no longer exists)

Requires human judgment (the script flags mismatches, but can't verify quality):

  • L4 (Domain Knowledge) — never auto-set or auto-removed. The script prominently asks for your input when L4 blocks higher maturity. Requires verifying that business-specific references, templates, and architecture knowledge are present.
  • L3 quality — the script detects examples/ and references/ folders + common patterns, but can't verify the quality of examples or whether patterns are applied correctly.
  • L5 baseline — the script checks for 3+ eval scenarios, 10+/10+ trigger tests, and last_validated non-null in evals.json. It cannot verify if evals were actually executed against a true baseline.
  • Target level — a product decision about where the skill should be headed.
